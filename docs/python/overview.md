# Python Оркестратор: Высокоуровневый обзор

Слой управления SORA полностью реализован на языке Python. В отличие от Rust-ядра, скрупулезно экономящего микросекунды, Python-слой оперирует на уровне макро-задач: отрисовка интерфейса, ведение сессионных логов, управление машинным состоянием (FSM) и рендеринг конфигураций.

## Разделение процессов

Python не читает фреймы с сетевого интерфейса напрямую и не блокирует основной цикл (Event Loop) тяжелыми вычислениями. Все сетевые операции производятся в скомпилированном ядре `sora-core`.

Оркестратор только периодически опрашивает MPSC очереди событий через `EventReceiver.poll_high()`.

:::danger
**Strict Compliance Statement**: Долгосрочное логирование (Persistence) и ведение сессионной аналитики используется исключительно для легальной отчетности (Audit Trail) и интеграции с корпоративными SIEM-решениями. Собранные PCAP-дампы и PMKID/EAPOL-маркеры подлежат немедленному удалению после закрытия контракта с заказчиком по оценке уязвимости WIDS-систем.
:::

## Ключевые компоненты

### 1. AsyncIO & FSM (`controller/`)
Центральный мозговой центр системы — класс `AttackController`. Это "Строгий Конечный Автомат" (Strict Finite State Machine), описывающий легальный флоу аудита: от сканирования эфира (Scanning) до симуляции атаки (Attacking) и генерации отчета (Reporting). FSM гарантирует предсказуемость поведения и обязательную очистку (Cleanup) всех подпроцессов (hostapd/dnsmasq) при возникновении критических ошибок `AdapterError`.

### 2. Пользовательский интерфейс (TUI / CLI)
Поскольку SORA — это headless-утилита, рассчитанная на работу по SSH, графический интерфейс реализован прямо в терминале с использованием фреймворка **Textual**. 
Интерфейс спроектирован как React-компоненты: он работает асинхронно, мгновенно отправляет команды в ядро и обновляет UI-виджеты (DataTable), не прерывая опрос IPC шины.

### 3. Session Persistence (SQLite)
Вся телеметрия записывается в легковесную базу данных SQLite.
- Каждая новая операция — это `session_id`.
- Для интеграции с классическим Wireshark, Python-слой записывает `pcapng_offset` (байтное смещение фрейма в PCAP-файле), чтобы аналитик по ИБ мог в 1 клик перейти к нужному кадру прямо из TUI.
- Учетные данные и хэши сохраняются внутри таблиц `credentials` и агрегируются в единый JSON Reporting.

### 4. Privilege Drop
Слой Python запускается от прав **root**, но сразу после инициализации raw-сокетов ядром и спавна всех нужных демонов (hostapd, dnsmasq), оркестратор использует `nix::unistd::setuid` для принудительного понижения своих прав в целях информационной безопасности (Least Privilege Principle).

Подробнее о конкретных узлах смотрите в соответствующих разделах документации.
