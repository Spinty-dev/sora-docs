# Менеджер Конфигураций (Config Manager)

`ConfigManager` — это компонент Python-слоя, отвечающий за парсинг TOML-профилей и динамическую генерацию конфигурационных файлов для сторонних демонов (`hostapd`, `dnsmasq`).

Главный принцип работы менеджера — **Single Source of Truth** (Единый источник истины).

## Принцип единого источника истины

В SORA v4.4 мы полностью устранили дублирование параметров между секциями TOML. Раньше параметры `channel` могли встречаться и в настройках атаки, и в шаблоне `hostapd`. Теперь:

1. Канонические значения (например, `our_channel`, `beacon_interval_tu`) хранятся в корневой секции атаки (например, `[attack.evil_twin]`).
2. Вложенные секции, такие как `[attack.evil_twin.hostapd]`, больше не содержат этих полей.
3. Если пользователь добавит `channel` в секцию `hostapd`, `ConfigManager` выдаст `WARNING` и проигнорирует его, подставив значение из родительской секции.

Это предотвращает ситуации, когда `hostapd` пытается запуститься на одном канале, в то время как `BeaconCloner` в Rust-ядре заблокировал другой.

## Динамический рендеринг и SIGHUP

При переходе FSM в состояние `evil_twin_ready`, `ConfigManager` получает объект с распарсенными Information Elements (IE) от легитимной точки доступа.

1. **Генерация**: Создается временный `hostapd.conf` в `/tmp/sora/`.
2. **Инъекция параметров**: Менеджер подставляет реальные `ssid`, `rsn_pairwise`, `ht_capab` и другие параметры, скопированные у цели.
3. **Запуск/Обновление**:
   - Если `hostapd` еще не запущен — он стартует.
   - Если `BeaconCloner` присылает событие `beacon_ie_changed` (оригинальная AP изменила настройки), менеджер перегенерирует конфиг и посылает процессу сигнал **`SIGHUP`**.

```python
def reload_hostapd(self, ie_data):
    conf_content = self.render_template(self.base_config, ie_data)
    with open(self.tmp_conf_path, 'w') as f:
        f.write(conf_content)
    
    if self.process:
        # Пытаемся перезагрузить конфиг без разрыва ассоциаций
        os.kill(self.process.pid, signal.SIGHUP)
    else:
        self.start_hostapd()
```

> [!IMPORTANT]
> Перезагрузка через `SIGHUP` занимает менее 200 мс, что позволяет сохранять подключение клиентов к "злому двойнику" даже при активной смене параметров оригинальной сети.

## Юридическая безопасность и шаблоны

Все шаблоны конфигураций, используемые `ConfigManager`, по умолчанию настроены на минимально необходимый функционал для проведения аудита. Мы намеренно избегаем автоматического включения функций перехвата трафика без явного указания в TOML-профиле, чтобы оператор осознавал каждое действие и его легальные последствия.
