# Persistence: SQLite & Pcapng Offsets

Этот раздел описывает механизмы долгосрочного хранения данных в SORA. Система объединяет реляционную базу данных (SQLite) для метаданных и структурированный бинарный лог (Pcapng) для радиочастотных фреймов.

## 1. Спецификация схемы SQLite (MetadataDB)

SORA использует SQLite для обеспечения целостности данных при резком отключении питания или аппаратных сбоях.

### Таблица `sessions`
Основная таблица сессий аудита.
- `id`: Первичный ключ.
- `pcapng_path`: Абсолютный путь к файлу дампа.
- `profile_name`: Имя TOML-профиля, использованного при старте.

### Таблица `handshakes`
Связующее звено между базой и дампом пакетов.
- `bssid`: MAC-адрес целевой точки доступа.
- **`pcapng_offset`**: Точное байтовое смещение начала блока EPB (Enhanced Packet Block) в pcapng-файле. Позволяет мгновенно извлечь конкретный хэндшейк без полного сканирования дампа.
- `captured_at`: ISO8601 метка времени захвата.

### Таблица `sae_captures` (Phase 4)
Специфичная таблица для WPA3-SAE (Dragonfly) аудита. Содержит BLOB-поля для коммитов и подтверждений (Commit/Confirm), необходимых для офлайн-атаки через `hashcat`.

## 2. Математика Pcapng Offsets

Для эффективной интеграции с инструментами анализа (Wireshark, `tcpdump`), SORA записывает данные в формате Pcapng (RFC 802.11).

### Структура и смещения блоков
При инициализации `PcapWriter` (см. `writer.rs:L155`) записываются два обязательных заголовка:
1. **SHB (Section Header Block)**: Фиксированный размер **28 байт**.
2. **IDB (Interface Description Block)**: Фиксированный размер **20 байт**.
   - *Base Offset*: Любой пакет EPB будет иметь смещение не менее **48 байт**.

### Расчет смещения EPB (Enhanced Packet Block)
Размер каждого блока EPB вычисляется по формуле:
$$BlockLength = 32 + ((DataLength + 3) \& !3) + 4$$
Где:
- `32`: Статичный заголовок EPB (Тип, Длина, Interface ID, Timestamp, Captured Len, Original Len).
- `((DataLength + 3) & !3)`: Длина данных фрейма с выравниванием по 4 байта (Padding).
- `4`: Дублирующее поле длины в конце блока (согласно спецификации Pcapng).

### Пример
Если захвачен EAPOL-фрейм длиной 121 байт:
1. Выравнивание: `(121 + 3) & !3 = 124`.
2. Длина блока: `32 + 124 + 4 = 160 байт`.
3. Смещение следующего пакета увеличится на 160.

## 3. Интеграция с Wireshark

Благодаря сохранению `pcapng_offset` в SQLite, SORA поддерживает мгновенный переход к пакету:
```bash
# Команда, генерируемая TUI для открытия конкретного хэндшейка
wireshark -r session.pcapng -Y "frame.offset == <pcapng_offset>"
```

> [!CAUTION]  
> **Strict Technical Note**: Прямая запись смещений требует отсутствия сжатия (compression) в Pcapng-файле. SORA записывает "сырой" поток EPB для обеспечения предсказуемости смещений, что увеличивает объем дискового пространства, но гарантирует O(1) доступ к уликам аудита.
