# Управление Интерфейсом (Netlink 802.11)

Модуль `nl80211` предоставляет абстракцию для безопасного и быстрого управления беспроводными адаптерами через семейство протоколов Netlink Linux.

Основной интерфейс взаимодействия определяется трейтом `InterfaceController`, что позволяет прозрачно применять Mock-объекты в юнит-тестах.

```rust
// core/src/nl80211/controller.rs
pub trait InterfaceController: Send + Sync {
    fn set_monitor_mode(&self, iface: &str) -> Result<(), Nl80211Error>;
    fn set_managed_mode(&self, iface: &str) -> Result<(), Nl80211Error>;
    fn set_channel(&self, iface: &str, channel: u8) -> Result<(), Nl80211Error>;
    fn set_tx_power(&self, iface: &str, dbm: u32) -> Result<(), Nl80211Error>;
    fn get_channel(&self, iface: &str) -> Result<u8, Nl80211Error>;
    fn interface_up(&self, iface: &str) -> Result<(), Nl80211Error>;
    fn interface_down(&self, iface: &str) -> Result<(), Nl80211Error>;
}
```

Модуль содержит две реализации этого трейта: `Nl80211Controller` (Fallback через `std::process::Command` к утилитам `ip` и `iw`) и основную высокопроизводительную реализацию `NeliController`.

## Neli Backend (`neli_backend.rs`)

Прямое взаимодействие с Linux Kernel Netlink (без вызова внешних bash-процессов) критично для **быстрого Channel Hopping** (сканирования каналов). Запуск бинарника `iw` на каждый прыжок создавал бы недопустимые задержки (около 15-30мс на вызов).

Используется крейт `neli` для парсинга и генерации бинарных сообщений Netlink.

### 1. Подключение и Resolving
При инициализации бэкенда происходит установка сокета `NlFamily::Generic` и динамический резолвинг ID семейства `nl80211`:
```rust
let mut socket = NlSocketHandle::connect(NlFamily::Generic, None, &[])
    .map_err(|e| /* ... */)?;

let family_id = socket.resolve_genl_family("nl80211")
    .map_err(|e| /* ... */)?;
```

### 2. Смена режима работы (Monitor Mode)
Для перевода карты в режим аудита, интерфейс сначала отключается (DOWN), затем отправляется сообщение `NL80211_CMD_SET_INTERFACE` с атрибутом `NL80211_ATTR_IFTYPE` равным `NL80211_IFTYPE_MONITOR` (6).
```rust
attrs.push(
    Nlattr::new(false, false, NL80211_ATTR_IFTYPE, NL80211_IFTYPE_MONITOR)?
);
self.send_command(iface, NL80211_CMD_SET_INTERFACE, attrs)?;
```

### 3. Низкоуровневый IOCTL (UP / DOWN)
Поскольку не все драйверы корректно обрабатывают UP/DOWN через nl80211, SORA напрямую обращается к ядру через `libc::ioctl`, манипулируя флагом `IFF_UP`.

```rust
// interface_up fallback
let sock = libc::socket(libc::AF_INET, libc::SOCK_DGRAM, 0);
libc::ioctl(sock, libc::SIOCGIFFLAGS, &mut ifr);
ifr.ifr_ifru.ifru_flags = ifr.ifr_ifru.ifru_flags as i16 | libc::IFF_UP as i16;
libc::ioctl(sock, libc::SIOCSIFFLAGS, &ifr);
libc::close(sock);
```

Эта гибридная архитектура (Netlink + IOCTL) гарантирует максимальную совместимость с самыми капризными Wi-Fi адаптерами (Qualcomm Atheros, Realtek).
