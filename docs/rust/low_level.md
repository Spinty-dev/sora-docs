# Low-Level Control (HW & Security)

Этот раздел описывает, как SORA взаимодействует с оборудованием и управляет правами доступа в системе Linux.

---

## 1. Контроллер Интерфейса (`nl80211/controller.rs`)

Для управления Wi-Fi адаптером (переключение каналов, смена режимов) SORA использует трейт `InterfaceController`. Это позволяет гибко подменять реальное железо на "заглушки" (mocks) во время тестов.

### Реализация `Nl80211Controller` (Phase 1):
На текущем этапе Rust вызывает системные утилиты `iw` и `ip` через `std::process::Command`. Это гарантирует совместимость с большинством дистрибутивов без сложной настройки Netlink библиотек.

*   **`set_monitor_mode`**: Переводит карту в режим монитора. Порядок действий: `ip link set down` -> `iw dev set type monitor` -> `ip link set up`.
*   **`set_channel`**: Устанавливает частоту захвата. Rust жестко контролирует этот процесс, чтобы `PacketEngine` не терял кадры при перескоке.
*   **`set_tx_power`**: Позволяет ограничить или усилить мощность сигнала для инъекций.

*В Phase 4 планируется переход на нативный Netlink протокол через библиотеку `neli`, что уберет зависимость от установленных в системе утилит `iw/ip`.*

## 2. Безопасность и сброс прав (`priv_drop.rs`)

SORA требует прав `root` или `CAP_NET_RAW` для открытия сырых сокетов. Однако держать весь процесс под суперпользователем — плохая практика безопасности (Security Risk).

### Механизм `drop_privileges`:
Модуль `priv_drop` позволяет SORA безопасно понизить свои привилегии сразу после того, как все сетевые ресурсы захвачены.

1.  **`DropTarget::from_env()`**: Автоматически определяет `UID/GID` пользователя, который запустил SORA через `sudo` (читает переменные `SUDO_UID` и `SUDO_GID`).
2.  **`setgroups` / `setgid` / `setuid`**: Процесс переходит в контекст обычного пользователя.
3.  **Верификация**: Rust проверяет `getuid()`. Если после попытки сброса процесс всё еще `root` — программа немедленно аварийно завершается (`StillRoot` error), так как безопасность превыше всего.

## 3. Запись дампов PCAP (`pcap/writer.rs`)

SORA не просто пишет байты в файл, она формирует валидный поток в формате **PCAPng**.

*   **`PcapHeader`**: При инициализации файла записывается "магическое число" `0xA1B2C3D4` и метаданные (версия PCAP, таймзоны, тип инкапсуляции 802.11).
*   **Zero-Copy Logic**: Благодаря использованию `crossbeam::ArrayQueue`, пакеты передаются между потоком захвата и потоком записи без лишнего копирования буферов в памяти.
*   **Async Sync**: Метод `shutdown()` гарантирует, что все пакеты из очереди будут сброшены на диск (flushed) перед завершением программы.
