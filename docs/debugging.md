# Отладка и Траблшутинг (Kernel Level)

Документация профессионального инструмента аудита была бы неполной без раздела по глубокой диагностике. Выход за пределы Python-интерфейса и понимание системных вызовов — ключевой навык для разработчика SORA.

## 1. Системная диагностика (strace)

`strace` — это основной инструмент для понимания того, почему SORA не видит пакеты или не может инициализировать интерфейс.

### Диагностика сетевых вызовов
Чтобы увидеть, какие именно пакеты и события приходят в `AF_PACKET` сокет:
```bash
# -e network: фильтровать только сетевые вызовы
# -s 64: показывать первые 64 байта данных (обычно хватает для заголовка 802.11)
sudo strace -p $(pgrep sora) -f -e network -s 64
```

### Типичная проблема: Пропажа пакетов
Если интерфейс в Monitor Mode, но `strace` не показывает вызовы `recvfrom` или они возвращают `0`:
- **Диагностика**: Кратковременно запустите `tcpdump -i wlan0mon -n`. Если `tcpdump` видит пакеты, а SORA нет — проблема в `sockaddr_ll` привязке (см. `af_packet.rs`).
- **Конфликт nl80211**: Если `strace` показывает `EBUSY` на вызовах `sendto` в Netlink-сокет — проверьте наличие процесса `wpa_supplicant`. 

> [!TIP]  
> **Chairman's Tip**: Используйте `strace -e ioctl`, если интерфейс не переходит в UP. Ищите ошибку `EPERM` — это верный признак того, что `capabilities` не проброшены в Python-процесс.

## 2. Отладка Rust-ядра через GDB

Поскольку Rust-ядро SORA загружается как динамическая библиотека (через PyO3) в процесс Python, отладка требует особого подхода.

### Привязка к процессу
1. Запустите SORA в обычном режиме.
2. В другом терминале найдите PID: `ps aux | grep sora`.
3. Запустите GDB:
```bash
sudo gdb -p <PID>
(gdb) directory core/src
(gdb) break packet_engine.rs:122
(gdb) continue
```

При захвате пакета GDB остановит выполнение. Вы сможете исследовать состояние `ParsedFrame` напрямую в памяти.

## 3. Hardware & Driver Quirks

Различные чипсеты ведут себя по-разному при работе с `nl80211`.

### Проверка Monitor Mode
Если переключение в Monitor Mode через Netlink проходит успешно, но пакеты не приходят, проверьте тип аппаратного интерфейса:
```bash
iw dev wlan0mon info
```
Если тип `managed`, значит драйвер проигнорировал команду `SET_INTERFACE`. 
- **Решение**: SORA автоматически пробует `IOCTL fallback`, но если и он не помогает — требуется ручная установка режима через `airmon-ng start`.

### Конфликты каналов (Channel Lock)
Если SORA заблокирована на одном канале, но физически интерфейс скачет по частотам:
- Проверьте `dmesg | grep nl80211`.
- Некоторые драйверы Intel (iwlwifi) не позволяют жестко фиксировать канал, если запущен `NetworkManager`. Используйте `airmon-ng check kill`.

> [!IMPORTANT]  
> **Strict Technical Detail**: Всегда проверяйте версию ядра `uname -a`. SORA оптимизирована для ядер 5.15+, где исправлены многие баги Netlink атрибутов для 802.11ax (Wi-Fi 6).
