# Архитектура и Жизненный Цикл Плагинов

Система плагинов SORA спроектирована с упором на максимальную изоляцию, стабильность ядра и языковую независимость. В отличие от монолитных архитектур, где плагины работают в том же адресном пространстве, что и основное приложение, SORA использует **модель раздельных процессов (Process-based Isolation)**.

## Принципы Изоляции

1. **Отсутствие GIL (для Python)**: Плагины на Python работают в собственных интерпретаторах, что позволяет им выполнять тяжелые вычисления или блокирующие I/O операции, не притормаживая основной `asyncio` цикл SORA.
2. **Crash Resilience**: Если плагин падает (Segfault в C++ или Unhandled Exception в Python), основное ядро SORA продолжает работу, фиксируя лишь событие `plugin_terminated`.
3. **Любой язык программирования**: Любой исполняемый файл, способный читать из `stdin` и писать в `stdout`, может быть плагином SORA.

## Жизненный цикл и Privilege Spawn

Одной из критических особенностей SORA является механизм управления привилегиями. Поскольку SORA после инициализации сырых сокетов сбрасывает права до обычного пользователя (`privilege drop`), плагины, требующие `root` (например, для настройки `iptables` или запуска `dnsmasq`), должны быть запущены **ДО** этого момента.

### Этапы запуска:
1. **Инициализация Ядра (Root)**: Открытие интерфейсов, создание RAW ловушек.
2. **Plugin Manager Spawn (Root)**: Менеджер читает секцию `[plugins]` в TOML и запускает процессы плагинов через `subprocess.Popen`. Плагины наследуют права `root`.
3. **Privilege Drop (SORA Core)**: Основной процесс SORA переходит в режим обычного пользователя.
4. **Активная Фаза**: Плагины обмениваются сообщениями с SORA через NDJSON шину.
5. **TearDown (Cleanup)**: При закрытии сессии SORA отправляет сигнал `SIGTERM` всем дочерним процессам. Плагины обязаны корректно завершиться и очистить свои временные ресурсы (например, правила файрвола).

## Конфигурация в TOML

Плагины активируются в главном файле профиля:

```toml
[plugins]
# Включение встроенных или внешних плагинов
enabled = ["captive_portal", "telegram_notify", "auto_crack"]

[plugins.captive_portal]
interface = "wlan1"
template = "dark_material"
redirect_all = true

[plugins.telegram_notify]
token = "ENV:TELEGRAM_TOKEN"
chat_id = "123456789"
```

:::danger
**Strict Compliance Statement**: Плагины, выполняющие активное воздействие (Captive Portal, Deauth-акторы), должны использоваться исключительно в рамках авторизованного аудита безопасности. Разработчик SORA не несет ответственности за код сторонних плагинов и их влияние на целевую инфраструктуру.
:::
