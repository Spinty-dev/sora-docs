nl80211 & Hybrid Interface Control

Этот раздел описывает архитектуру управления Wi-Fi адаптером в SORA. Переключение режимов (Managed/Monitor) и смена частот реализованы через гибридный механизм, сочетающий современный протокол Generic Netlink (nl80211) и классические системные вызовы IOCTL.

1. Спецификация Netlink (nl80211)​

SORA использует библиотеку neli для формирования сообщений Generic Netlink. Это позволяет изменять параметры радиомодуля без разрыва существующих соединений (там, где это поддерживает драйвер).

Основные команды и атрибуты (neli_backend.rs:L16)​

КонстантаЗначениеОписание (Linux Kernel Documentation)
NL80211_CMD_SET_INTERFACE6Изменение типа интерфейса (monitor/station)
NL80211_CMD_SET_CHANNEL31Перестройка синтезатора частоты (Channel Hopping)
NL80211_ATTR_IFINDEX3Системный индекс интерфейса
NL80211_ATTR_WIPHY_FREQ38Целевая частота в МГц
NL80211_ATTR_IFTYPE5Тип интерфейса (использует enum nl80211_iftype)

Механизм смены частоты (Frequency Calculation)​

SORA автоматически рассчитывает частоту в МГц из номера канала, чтобы избежать неоднозначности между 2.4 GHz и 5 GHz диапазонами:

  • 2.4 GHz: freq = 2407 + (channel * 5) (стандарт IEEE 802.11 §15.4.6.2)
  • 5 GHz: freq = 5000 + (channel * 5) (стандарт IEEE 802.11 §E.1)

2. IOCTL Fallback: Управление состоянием (UP/DOWN)​

Хотя Netlink позволяет управлять состоянием интерфейса, на многих драйверах (особенно ath9k и rtl8188) изменение типа интерфейса (например, с Managed на Monitor) требует обязательного перевода интерфейса в состояние DOWN.

SORA использует системный вызов ioctl с флагами SIOCSIFFLAGS для обеспечения максимальной совместимости.

Алгоритм переключения режима (neli_backend.rs:L112)​

  1. interface_down:
    • Вызов libc::socket(AF_INET, SOCK_DGRAM, 0).
    • Загрузка текущих флагов через SIOCGIFFLAGS.
    • Бтовая операция: flags &= !IFF_UP.
    • Применение через SIOCSIFFLAGS.
  2. Netlink SET_INTERFACE:
    • Посылка сообщения NL80211_CMD_SET_INTERFACE.
    • Атрибут NL80211_ATTR_IFTYPE = NL80211_IFTYPE_MONITOR.
  3. interface_up:
    • Бтовая операция: flags |= IFF_UP.
    • Применение через SIOCSIFFLAGS.

3. Анализ надежности и типичные ошибки​

Иерархия управления в SORA спроектирована так, чтобы минимизировать риск "зависания" интерфейса в неопределенном состоянии.

[!WARNING]
Strict Technical Note: Некоторые драйверы возвращают EBUSY при попытке смены частоты через Netlink, если интерфейс занят другим процессом (например, NetworkManager). SORA рекомендует использовать airmon-ng check kill перед запуском, чтобы избежать конфликтов на уровне Netlink-сокетов.

Приоритеты управления:​

  1. Netlink: Высокоскоростное управление без перезагрузки стека.
  2. IOCTL: Гарантированное изменение состояния уровня L2.
  3. Process Fallback: (В разработке) Вызов утилит iw / ip в случае блокировки Netlink-сокетов родительским процессом.
