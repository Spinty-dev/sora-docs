# nl80211 与混合接口控制

本章节介绍了 SORA 中 Wi-Fi 适配管理架构。模式切换（托管/监听）和频率更改是通过结合现代 **Generic Netlink (nl80211)** 协议和经典 **IOCTL** 系统调用的混合机制实现的。

## 1. Netlink 规范 (nl80211)

SORA 使用 `neli` 库来构建 Generic Netlink 消息。这允许在不中断现有连接或重新加载驱动程序的情况下更改无线电模块参数（在驱动程序支持的情况下）。

### 核心命令与属性 (neli_backend.rs:L16)

| 常量 | 值 | 描述 (Linux 内核文档) |
| :--- | :--- | :--- |
| `NL80211_CMD_SET_INTERFACE` | 6 | 更改接口类型 (监听/站) |
| `NL80211_CMD_SET_CHANNEL` | 31 | 重新调整频率频率合成器 (信道跳频) |
| `NL80211_ATTR_IFINDEX` | 3 | 系统接口索引 |
| `NL80211_ATTR_WIPHY_FREQ` | 38 | 目标频率 (MHz) |
| `NL80211_ATTR_IFTYPE` | 5 | 接口类型 (使用 `enum nl80211_iftype`) |

### 频率计算机制 (Frequency Calculation)

SORA 会根据信道号自动计算 MHz 为单位的频率，以避免 2.4 GHz 和 5 GHz 频段之间的歧义：
- **2.4 GHz**: `freq = 2407 + (channel * 5)` (标准 IEEE 802.11 §15.4.6.2)
- **5 GHz**: `freq = 5000 + (channel * 5)` (标准 IEEE 802.11 §E.1)

## 2. IOCTL 回退：状态管理 (UP/DOWN)

尽管 Netlink 允许管理接口状态，但在许多驱动程序（特别是 `ath9k` 和 `rtl8188`）上，更改接口类型（例如从 `Managed` 切换到 `Monitor`）必须先将接口设为 `DOWN` 状态。

SORA 使用带有 `SIOCSIFFLAGS` 标志的 `ioctl` 系统调用，以确保最大的兼容性。

### 模式切换算法 (neli_backend.rs:L112)

1. **`interface_down`**：
   - 调用 `libc::socket(AF_INET, SOCK_DGRAM, 0)`。
   - 通过 `SIOCGIFFLAGS` 加载当前标志。
   - 位操作：`flags &= !IFF_UP`。
   - 通过 `SIOCSIFFLAGS` 应用。
2. **Netlink `SET_INTERFACE`**：
   - 发送 `NL80211_CMD_SET_INTERFACE` 消息。
   - 属性 `NL80211_ATTR_IFTYPE = NL80211_IFTYPE_MONITOR`。
3. **`interface_up`**：
   - 位操作：`flags |= IFF_UP`。
   - 通过 `SIOCSIFFLAGS` 应用。

## 3. 可靠性分析与常见错误

SORA 的控制层级旨在最大限度地减少接口在未定义状态下“挂起”的风险。

:::warning
**严格的技术说明**：如果接口被其他进程（例如 `NetworkManager`）占用，某些驱动程序在尝试通过 Netlink 更改频率时会返回 `EBUSY`。SORA 建议在启动前运行 `airmon-ng check kill`，以避免 Netlink 套接字层级的冲突。
:::

### 控制优先级：
1. **Netlink**：不重新加载协议栈的高速控制。
2. **IOCTL**：保证 L2 层级状态更改。
3. **进程回退 (Process Fallback)**：（开发中）在 Netlink 套接字被父进程锁定时调用 `iw` / `ip` 工具。
