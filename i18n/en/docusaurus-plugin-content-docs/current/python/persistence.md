# Persistence: SQLite & Pcapng Offsets

This section describes the long-term data storage mechanisms in SORA. The system combines a relational database (SQLite) for metadata and a structured binary log (Pcapng) for radio frequency frames.

## 1. SQLite Schema Specification (MetadataDB)

SORA uses SQLite to ensure data integrity during sudden power outages or hardware failures.

### `sessions` Table
The main table for audit sessions.
- `id`: Primary key.
- `pcapng_path`: Absolute path to the dump file.
- `profile_name`: Name of the TOML profile used at startup.

### `handshakes` Table
The link between the database and the packet dump.
- `bssid`: MAC address of the target access point.
- **`pcapng_offset`**: Exact byte offset of the start of the EPB (Enhanced Packet Block) in the pcapng file. Allows for instantaneous extraction of a specific handshake without a full dump scan.
- `captured_at`: ISO8601 timestamp of the capture.

### `sae_captures` Table (Phase 4)
A specific table for WPA3-SAE (Dragonfly) auditing. Contains BLOB fields for commits and confirmations (Commit/Confirm) required for offline attacks via `hashcat`.

## 2. Pcapng Offsets Mathematics

For efficient integration with analysis tools (Wireshark, `tcpdump`), SORA records data in Pcapng format (RFC 802.11).

### Block Structure and Offsets
When `PcapWriter` is initialized (see `writer.rs:L155`), two mandatory headers are written:
1. **SHB (Section Header Block)**: Fixed size of **28 bytes**.
2. **IDB (Interface Description Block)**: Fixed size of **20 bytes**.
   - *Base Offset*: Any EPB packet will have an offset of at least **48 bytes**.

### Calculating EPB (Enhanced Packet Block) Offset
The size of each EPB block is calculated using the formula:
```text
BlockLength = 32 + ((DataLength + 3) & ~3) + 4
```
Where:
- `32`: Static EPB header (Type, Length, Interface ID, Timestamp, Captured Len, Original Len).
- `((DataLength + 3) & ~3)`: Frame data length with 4-byte alignment (Padding).
- `4`: Duplicate length field at the end of the block (according to the Pcapng specification).

### Example
If an EAPOL frame with a length of 121 bytes is captured:
1. Alignment: `(121 + 3) & ~3 = 124`.
2. Block Length: `32 + 124 + 4 = 160 bytes`.
3. The next packet's offset will increase by 160.

## 3. Wireshark Integration

By storing the `pcapng_offset` in SQLite, SORA supports near-instantaneous jumping to a packet:
```bash
# Command generated by the TUI to open a specific handshake
wireshark -r session.pcapng -Y "frame.offset == <pcapng_offset>"
```

:::danger
**Strict Technical Note**: Direct offset recording requires the absence of compression in the Pcapng file. SORA writes a "raw" EPB stream to ensure offset predictability, which increases disk space usage but guarantees O(1) access to audit evidence.
:::
