# nl80211 & Hybrid Interface Control

This section describes the architecture of Wi-Fi adapter management in SORA. Mode switching (Managed/Monitor) and frequency changes are implemented through a hybrid mechanism that combines the modern **Generic Netlink (nl80211)** protocol and classic **IOCTL** system calls.

## 1. Netlink Specification (nl80211)

SORA uses the `neli` library to form Generic Netlink messages. This allows changing radio module parameters without dropping existing connections (where supported by the driver).

### Core Commands and Attributes (neli_backend.rs:L16)

| Constant | Value | Description (Linux Kernel Documentation) |
| :--- | :--- | :--- |
| `NL80211_CMD_SET_INTERFACE` | 6 | Change interface type (monitor/station) |
| `NL80211_CMD_SET_CHANNEL` | 31 | Retune the frequency synthesizer (Channel Hopping) |
| `NL80211_ATTR_IFINDEX` | 3 | System interface index |
| `NL80211_ATTR_WIPHY_FREQ` | 38 | Target frequency in MHz |
| `NL80211_ATTR_IFTYPE` | 5 | Interface type (uses `enum nl80211_iftype`) |

### Frequency Calculation Mechanism

SORA automatically calculates the frequency in MHz from the channel number to avoid ambiguity between the 2.4 GHz and 5 GHz bands:
- **2.4 GHz**: `freq = 2407 + (channel * 5)` (IEEE 802.11 §15.4.6.2 standard)
- **5 GHz**: `freq = 5000 + (channel * 5)` (IEEE 802.11 §E.1 standard)

## 2. IOCTL Fallback: State Management (UP/DOWN)

Although Netlink allows managing interface state, on many drivers (especially `ath9k` and `rtl8188`), changing the interface type (e.g., from `Managed` to `Monitor`) requires taking the interface `DOWN`.

SORA uses the `ioctl` system call with `SIOCSIFFLAGS` flags to ensure maximum compatibility.

### Mode Switching Algorithm (neli_backend.rs:L112)

1. **`interface_down`**:
   - `libc::socket(AF_INET, SOCK_DGRAM, 0)` call.
   - Load current flags via `SIOCGIFFLAGS`.
   - Bitwise operation: `flags &= !IFF_UP`.
   - Apply via `SIOCSIFFLAGS`.
2. **Netlink `SET_INTERFACE`**:
   - Send `NL80211_CMD_SET_INTERFACE` message.
   - Attribute `NL80211_ATTR_IFTYPE = NL80211_IFTYPE_MONITOR`.
3. **`interface_up`**:
   - Bitwise operation: `flags |= IFF_UP`.
   - Apply via `SIOCSIFFLAGS`.

## 3. Reliability Analysis and Common Errors

The control hierarchy in SORA is designed to minimize the risk of the interface "hanging" in an undefined state.

:::warning
**Strict Technical Note**: Some drivers return `EBUSY` when attempting to change frequency via Netlink if the interface is occupied by another process (e.g., `NetworkManager`). SORA recommends using `airmon-ng check kill` before launch to avoid conflicts at the Netlink socket level.
:::

### Control Priorities:
1. **Netlink**: High-speed control without reloading the stack.
2. **IOCTL**: Guaranteed change of L2 state.
3. **Process Fallback**: (In development) Calling `iw` / `ip` utilities in case of Netlink socket blockage by a parent process.
